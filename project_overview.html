项目能做什么

用户发消息（Telegram）
↓
bot.py 接收
↓
agent（LangGraph图）决策
├── 直接回答 ──→ 调 DeepSeek
├── 需要搜索 ──→ Tavily → 结果给 DeepSeek
└── 需要查记忆 → Qdrant 向量检索 → 相关记忆给 DeepSeek
↑
每次对话结束：新内容向量化存入 Qdrant
记忆超量时：DeepSeek 压缩整合旧记忆
─────────────────
【主动推送（月5新增）】
scheduler.py 每日定时触发
↓
查询 Qdrant 近期记忆
↓
DeepSeek 判断"今天有没有值得提醒用户的事"
↓（有则发，没有则跳过）
bot.py 主动推送到用户 Telegram

项目文件架构

第一类 应该能自己写出骨架 第二类 深度理解流程，AI写库调用 第三类 理解目的，AI写实现 
my-agent/
├── [第一类] config.py            ← 读取所有 API key 和配置
├── [第一类] llm.py               ← 封装 DeepSeek API 调用（async）
├── [第二类] bot.py               ← Telegram 收发消息
├── [第二类] agent.py             ← 手写 agent 循环（月2临时版）
├── [第二类] session.py           ← 当前对话短期记忆
├── tools/
│   ├── [第一类] web_search.py    ← 调 Tavily 搜索
│   └── [第一类] memory_recall.py ← 查询向量数据库，返回相关记忆
├── memory/
│   ├── [第一类] embed.py         ← 文字 → 向量
│   ├── [第三类] store.py         ← 向量数据库增删查
│   ├── [第三类] search.py        ← 相似度检索 + Rerank
│   └── [第三类] consolidate.py   ← 记忆超量时 LLM 压缩
├── agent/               ← 月3用 LangGraph 正式重写
│   ├── [第一类] state.py         ← LangGraph 状态定义（TypedDict）
│   ├── [第三类] graph.py         ← LangGraph 图结构（节点+条件路由）
│   └── [第二类] nodes.py         ← 每个节点的具体逻辑
├── eval/
│   └── [第二类] metrics.py       ← Recall@5 评测脚本
├── [第三类] scheduler.py         ← APScheduler 定时：每日查记忆，LLM 决定是否主动推送
└── main.py              ← 启动入口
铁律：tools/ 永远不得 import agent/ 内容，否则产生循环依赖。

每日学习框架（怎么学 / 每天做什么）

四步公式（每次学新东西都用这个流程）

步骤做什么怎么知道完成了
Step 1 · 读读官方文档或教程中关于这个概念的最短解释（≤10分钟）能用自己的话说出"这个东西是干什么的"
Step 2 · 抄把最小可运行的示例代码完整手打一遍（不要复制粘贴）代码跑通，没有报错
Step 3 · 改改其中一个变量/参数，预测结果，然后运行，看对不对预测和结果一致；如果不一致，找到原因
Step 4 · 用把这个知识点用到项目文件里，哪怕只有 3 行项目代码里出现了这个知识点，而且能解释为什么这样写

每天最少学习量（工作日 / 休息日）

天类型时间必须完成如果有余力
工作日3-4 小时四步公式走完一个新知识点 + 把它写进项目多走一个知识点
休息日8 小时以上完成当周目标文件的第一个可运行版本写测试 / 清理代码 / 对照 akashic-agent 看一个文件

第 1 周每日安排示例（config.py 周）

天任务验收
Day 1学 python-dotenv：读文档 → 手打示例 → 改变量名 → 跑通终端打印出 .env 里的值
Day 2学 Python dataclass 或字典配置模式config.py 第一版写出来，能 from config import cfg
Day 3学 os.environ 与 .env 的区别；加默认值处理config.py 加上缺少 key 时的报错提示
Day 4和 Claude Code 对话：把你写的 config.py 贴进来，问"哪里不规范"至少找出 1 个改进点并修掉
Day 5（休息日）写 llm.py 第一版：调 DeepSeek API 返回一句话终端打印出 DeepSeek 的回复

卡住了怎么办

15分钟规则：卡住超过 15 分钟，立刻用 print 二分法定位问题（在哪行之前加 print 还正常，在哪行之后就出错）。定位到具体行后再问 Claude Code，把那一行代码 + 报错信息 + "我以为它会做什么" 一起贴给它。

不要做的事：不要先问"这个要怎么学"，先动手；不要一次读完整个文档，只读你现在需要的部分；不要跳过 Step 2 的手打步骤。

如何判断"学会了可以用"

能做到以下三件事就算学会：
① 不看文档，能写出这个知识点的最基本用法
② 能解释"为什么这里用这个而不是别的"
③ 遇到报错，知道去哪里找原因（官方文档 / 打 print / 看报错行号）

技术选型理由（面试必备）

技术替代方案选择理由接受的代价
DeepSeekGPT-4、Claude价格极低（约GPT-4的1/200），中文好，API格式兼容OpenAI复杂推理弱于GPT-4
ChromaDB（月3）→ Qdrant（月5）全程Qdrant、全程ChromaDBChromaDB 本地零部署加速开发迭代；月5迁 Qdrant 覆盖JD要求；迁移过程本身是面试素材需迁移一次，感受两者接口差异
TavilySerpAPI、DuckDuckGo专为LLM设计（返回干净文本），有免费额度较新，文档相对少
LangGraph纯手写、基础LangChainJD明确要求，结构化状态管理优于手写额外学习成本
python-telegram-botaiogram、telebot文档最全，社区最大，v20+原生支持async代码比aiogram略啰嗦
月2手写→月3 LangGraph重写直接用LangGraph / 全程手写月2手写理解tool loop原理（用nanobot/runner.py验证），月3用LangGraph重新实现——面试能解释两层：LangGraph的用法 + 它背后解决的问题月3有重写工作量，但比"月4迁移"更早用上LangGraph
APSchedulerCelery Beat、cron纯 Python 轻量级，不需要额外中间件，异步兼容好不适合分布式大规模任务

与 akashic-agent 对比

akashic-agent 有但你没有

  • QQ 渠道支持
  • Plugin 系统（10+ 插件）
  • Web Dashboard
  • 主动行为（RSS 推送）
  • Peer agent 协作

你有但 akashic-agent 没有

  • Recall@5 评测脚本（量化检索效果）
  • Rerank 前后对比数据
  • LangGraph 框架（它用自定义循环）
  • 每个设计决策的明确文档
  • 简单主动推送（scheduler.py，基于记忆决策）
面试叙述："相比 akashic-agent，我的项目更聚焦：加入了它没有的 Recall@5 评测体系，能量化说明记忆检索有效性；同时实现了简单版主动推送——区别在于 akashic-agent 的推送是全自动漂移系统，我的是基于记忆的 LLM 判断推送，设计更简单但更易理解。"
